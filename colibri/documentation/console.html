Console commands

  1. Introduction
  2. Framework related commands

Introduction

One of the most easiest way of managing Opis Colibri is through console commands. Console commands can be either provided by the framework or user defined. Here is a list with all the commands provided by the framework and a short description for each of them. You should be aware that Opis Colibri must be fully installed before being able to use any of the framework related commands.

Framework related commands

The list command

Obtain a list with all the supported commands.

Usage: php colibri list

The help command

Display informations about a specific command.

Usage: php colibri help <command-name>

The modules command

Obtain a list with all the available modules.

Usage: php colibri modules

The about command

Display detailed information about a specific module.

Usage: php colibri about <module-name>

The install command

Install all the specified modules. This command takes as arguments a list of one or more modules separated by space. If you use the --enable option, the modules will be enabled after they are installed.

Usage: php colibri install [--enable] <module-1>[ <module-2> <module-3> ... <module-1=n>]

The enable command

Enable all the specified modules. This command takes as arguments a list of one or more modules separated by space.

Usage: php colibri enable <module-1>[ <module-2> <module-3> ... <module-n>]

The disable command

Disable all the specified modules. This command takes as arguments a list of one or more modules separated by space. If you use the --uninstall option, the modules will be uninstalled after they are disabled.

Usage: php colibri disable [--uninstall] <module-1>[ <module-2> <module-3> ... <module-n>]

The uninstall command

Uninstall all the specified modules. This command takes as arguments a list of one or more modules separated by space.

Usage: php colibri uninstall <module-1>[ <module-2> <module-3> ... <module-n>]

The collect command

Collect all the resources declared by modules.

Usage: php colibri collect

Composer related commands

The get-module command

Use Composer to download a list of one or more modules. Each module in the list must be declared using the following format: vendor/module:version. If the version is omitted, * will be used as the module's version. If the vendor is ommitted, opis-colibri will be used as the module's vendor.

Usage: composer get-module vendor/example1:1.0.*
Usage: composer get-module vendor/example1:1.0.* vendor2/example2:3.0.*
Usage: composer get-module vendor/example
Usage: composer get-module module1
Usage: composer get-module mdoule1 vendor/example1:1.0.* module2

The get-package command

Use Composer to download all modules and libraries defined by an Opis Colibri package. The package must be declared using the following format: vendor/package. If the vendor is omitted opis-colibri will be used as the package's vendor. Optionally, you can pass a second parameter representing the package's version. If package's version is omitted, * will be used.

Usage: composer get-package vendor/pack1
Usage: composer get-package vendor/pack1 ~1.0
Usage: composer get-package pack1
Usage: composer get-package pack1 2.0.*
