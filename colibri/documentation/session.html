Session storages

Introduction

The framework's session manager is based on the Opis Session library and session storages are collected using the SessionStorages method. 

use Opis\Colibri\Define;

Define::SessionStorages(function($storage){

    // Collect session storages

});

You can override a session storage by specifying a lower collecting priority. 

use Opis\Colibri\Define;

Define::SessionStorages(function($storage){

    // Collect session storages

}, -1);

The collector passed to the callback function is an object that implements the Opis\Colibri\SessionCollectorInterface interface. 

interface StorageCollectorInterface
{
    /**
     * Register a new storage
     *
     * @param   string      $storage        Storage name
     * @param   \Closure    $constructor    Storage constructor callback
     * @param   boolean     $default        (optional) Default flag
     *
     * @return  mixed
     */
    
    public function register($storage, Closure $constructor, $default = false);
}


interface SessionCollectorInterface extends StorageCollectorInterface
{
    
}

Registering session storages

You can register a new session storage by passing the storage name and an anonymous callback function to the register method. The callback must return an instance of a class that implements the SessionHandlerInterface interface. 

use Opis\Colibri\Define;
use Opis\Session\Storage\File as FileStorage;

Define::SessionStorages(function($storage){

    $storage->register('foo', function(){
        return new FileStorage('/path/to/folder');
    });

});

You can designate a storage as being the default storage by passing true as the third argument to the register method. 

use Opis\Colibri\Define;
use Opis\Session\Storage\File as FileStorage;

Define::SessionStorages(function($storage){

    $storage->register('foo', function(){
        return new FileStorage('/path/to/folder');
    }, true);

});

Working with sessions

Important! Because of the way Opis Session works, using multiple session storages simultaneously is not recommended.

You can manage user sessions with the help of the Session global function. This function takes as arguments the name of the session storage you want to use to store session data. 

Session('foo')->set('user', 'Some value');

If the storage name is ommited, then the default storage will be used. 

Session()->set('user', 'Some value');
