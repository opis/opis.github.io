Introduction

Beside the fact that a module provides collectible items that can be used by the framework, each module behaves as an ordinary library and like any other library it can come with a set of classes that can be used by the module itself or by other modules.

Source folder

The source folder of a module is the folder where all module's classes must be added. By default the source folder is the same with the folder where the module.json file was added. Changing the source folder requires you to add a source property into the module.json file.

Module's namespace

The module's classes must be shipped under a namespace. By default, the namespace of a module is formed using the module's name and the Opis\Colibri\Module prefix. For example, if your module is named foo, then your module's namespace will be Opis\Colibri\Module\Foo. If your module is named foo-bar, then your module's namespace will be Opis\Colibri\Module\FooBar. You can set a custom namespace for your module by adding a namespace property into the module.json file.

The namespace is mapped to the source folder using the PSR-4 standard.
